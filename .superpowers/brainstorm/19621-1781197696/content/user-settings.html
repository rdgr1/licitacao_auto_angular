Estrutura de auditoria (super admin — futuro)

Qualquer mudança feita em preferências, perfil ou acesso é gravada numa tabela audit_log no backend com userId · timestamp · campo alterado · valor anterior · valor novo · IP. O super admin tem uma view de auditoria por usuário. A mesma estrutura serve para rastrear ações de leads (quem descartou, quem aprovou) num segundo momento.
